## 1. 概述

### 1.1 背景

金融数据公司咨询部门负责银行监管报送项目。BA 在访谈过程中需查阅大量制度文件、填报说明与口径资料，资料分散、检索困难。

### 1.2 产品定位

面向监管报送咨询团队的**本地离线资料检索工具**：输入关键词，快速定位各报送规范中相关数据项及其集市映射信息。

### 1.3 目标用户

| 角色 | 场景 |
|------|------|
| 一线 BA | 访谈准备、现场答疑、口径核对 |
| 项目经理 | 项目启动调研、引用准确性检查 |
| 新入职顾问 | 通过关键词学习监管口径 |

### 1.4 交付形态

| 阶段 | 形态 |
|------|------|
| Phase 1（MVP） | 网页端 + 本地部署 |
| Phase 2 | 桌面应用（Electron 打包，可离线） |

### 1.5 非目标（Out of Scope）

- AI 问答 / 大模型总结（远期可选）
- 多人协作编辑资料
- 与银行生产系统对接
- 资料版本审批流程
- 云端部署 / 外网依赖

## 2. 功能需求

### 2.1 页面清单

| 页面 | 路由（建议） | 优先级 |
|------|-------------|--------|
| 查询页（搜索 + 结果） | `/` | P0 |
| 资料导入管理 | `/import` | P0 |
| 系统设置 | `/settings` | P2 |

### 2.2 查询页 — 搜索框

| 编号 | 需求 |
|------|------|
| FR-01 | 提供关键词输入框与「搜索」按钮 |
| FR-02 | 支持 Enter 触发搜索 |
| FR-03 | 搜索范围为所有报送表的 `数据项名称` 字段（包含匹配） |
| FR-04 | 空关键词提交时提示「请输入搜索关键词」，不发起请求 |

### 2.3 查询页 — 联想（Autocomplete）

| 编号 | 需求 |
|------|------|
| FR-10 | 输入 ≥ 1 个字符时触发联想 |
| FR-11 | 匹配方式：包含匹配 `数据项名称` |
| FR-12 | 排序：相关度优先（完全匹配 > 前缀匹配 > 位置靠前 > 名称更短） |
| FR-13 | 每条展示：数据项名称、表名、报送项目名称 |
| FR-14 | 最多返回 10 条 |
| FR-15 | 点击联想项 → 以该数据项名称执行搜索 |
| FR-16 | 无匹配时显示「无匹配数据项」，不阻断用户继续搜索 |
| FR-17 | 联想响应时间 ≤ 300ms（万级数据） |

### 2.4 查询页 — 报送标签页

| 编号 | 需求 |
|------|------|
| FR-20 | 搜索后以标签页展示各报送项目结果 |
| FR-21 | 报送类型：一表通、EAST、金数、1104、大集中、征信、rcpmis、SAFE（可扩展） |
| FR-22 | **仅显示有命中结果的标签页**，无命中则不显示 |
| FR-23 | 默认选中命中数最多的报送 |
| FR-24 | 激活标签：主题色文字 + 底部指示线；非激活：灰色文字 |

### 2.5 查询页 — 结果展示

#### 2.5.1 分组规则

| 编号 | 需求 |
|------|------|
| FR-30 | 当前标签页内，按 `表名`（报表名称）分块展示 |
| FR-31 | 同一表名下所有命中数据项归入同一块 |
| FR-32 | 多个报表块之间保留垂直间距 |

#### 2.5.2 左右分栏布局

每个报表块使用**统一五列网格**，组头与数据行列线对齐。

**左侧 — 监管报送：**

| 位置 | 字段 | 数据库列 | 层级 |
|------|------|----------|------|
| 组头第 1 行 | 报表名称 | `表名` | 表级 |
| 组头第 2 行 | 采集范围 | `采集范围` | 表级 |
| 数据列 1 | 数据项名称 | `数据项名称` | 数据项级 |
| 数据列 2 | 数据元说明 | `数据元说明` | 数据项级 |

**右侧 — 集市：**

| 位置 | 字段 | 数据库列 | 层级 |
|------|------|----------|------|
| 组头 | 数据范围 | `取数表` + `关联条件` | 表级 |
| 数据列 3 | 表 | `集市来源表` | 数据项级 |
| 数据列 4 | 字段 | `集市来源字段` | 数据项级 |
| 数据列 5 | 逻辑 | `逻辑` | 数据项级 |

#### 2.5.3 列对齐规则

| 编号 | 需求 |
|------|------|
| FR-40 | 全表（组头 + 表头 + 数据行）共用同一 `grid-template-columns` |
| FR-41 | 「数据范围」标签占**第 3 列**，与下方「表」列左缘对齐 |
| FR-42 | 取数表、关联条件内容合并占**第 4–5 列**（跨字段列 + 逻辑列） |
| FR-43 | 组头左侧两行（报表名称、采集范围）占第 1–2 列，与数据项名称、数据元说明列对齐 |

#### 2.5.4 组头行合并

| 编号 | 需求 |
|------|------|
| FR-50 | 「数据范围」标签纵向合并 2 行（与报表名称、采集范围两行等高） |
| FR-51 | 取数表 / 关联条件内容区同样纵向合并 2 行 |

#### 2.5.5 边界状态

| 状态 | 表现 |
|------|------|
| 全部无命中 | 「未找到包含「xxx」的数据项」 |
| 加载中 | 搜索框旁 loading 指示 |
| 资料库为空 | 引导前往资料导入页 |

### 2.6 资料导入管理页

| 编号 | 需求 |
|------|------|
| FR-60 | 按报送类型切换导入目标（标签：一表通、EAST 等） |
| FR-61 | 支持 Excel / CSV 文件上传（拖拽或选择） |
| FR-62 | 导入前校验必填列是否存在 |
| FR-63 | 主键冲突策略：同 `表名` + `数据项名称` 覆盖 |

## 3. 数据模型

### 3.1 设计原则

- 每个监管报送独立一张数据表（字段规格不同）
- 通过 `report_config` 配置表驱动 UI 展示与扩展
- 所有报送表**必有字段**：`表名`、`数据项名称`、`集市来源表采集范围`

### 3.2 报送配置表 `report_config`

| 字段 | 类型 | 说明 |
|------|------|------|
| code | TEXT PK | 报送代码，如 `YBT` |
| name | TEXT | 报送名称，如 `一表通` |
| table_name | TEXT | 数据表名，如 `YBT_ZL` |
| search_field | TEXT | 搜索字段，默认 `数据项名称` |
| display_fields | TEXT | JSON 数组，展示字段列表 |
| sort_order | INTEGER | 标签页排序 |

### 3.3 一表通表 `YBT_ZL`（示例）

| 字段 | 层级 | 说明 | 约束 |
|------|------|------|------|
| 表名 | 表级 | 监管报表名称 | PK |
| 数据项名称 | 数据项级 | 字段/指标名称 | PK；搜索字段 |
| 采集范围 | 表级 | 该表的采集范围（同表名共用） | |
| 取数表 | 表级 | 集市取数来源表清单 | 组头右侧 |
| 关联条件 | 表级 | 表间关联逻辑 | 组头右侧 |
| 数据元说明 | 数据项级 | 数据元定义与口径 | |
| 集市来源表 | 数据项级 | UI 列名「表」 | |
| 集市来源字段 | 数据项级 | UI 列名「字段」 | |
| 集市来源表采集范围 | 数据项级 | 必有，MVP 不在 UI 展示 | |
| 逻辑 | 数据项级 | 取数/加工逻辑 | |

**主键**：`表名` + `数据项名称`

### 3.4 其他报送表

结构类似，字段因报送规格而异，但必有：`表名`、`数据项名称`、`集市来源表采集范围`。

MVP 先实现一表通；架构预留多报送扩展。

|-----------|-----------|
| 表名 | 表名 |
| 数据项名称 | 数据项名称 |

| 取数表 | 取数表 |

| 数据元说明 | 数据元说明 |
| 集市来源表 | 集市来源表 |

| 集市来源表采集范围 | 集市来源表采集范围 |
| 逻辑 | 逻辑 |

## 4. UI 样式规范

### 4.1 颜色

| 规则 | 说明 |
|------|------|
| 无分区着色 | 监管侧与集市侧使用相同背景色 |
| 无关键词高亮 | 匹配词不做蓝色或其他颜色标记 |
| 标签/表头底色 | `fill.secondary`（浅灰） |
| 数据行 | 交替 `editor` / `fill.tertiary`（斑马纹） |

### 4.2 单元格

| 属性 | 值 |
|------|-----|
| 内边距 | 10px 上下，12px 左右 |
| 最小行高 | 40px |
| 边框 | 1px 细线；组头与数据区之间 2px 分隔 |
| 标签/表头对齐 | 水平居中，垂直居中 |
| 内容对齐 | 左对齐，垂直居中 |
| 外框 | 1px 边框，圆角 6px |

### 4.3 五列宽度比例（建议）

```
grid-template-columns:
  minmax(108px, 1.05fr)   /* 数据项名称 / 报表名称标签 */
  minmax(128px, 1.25fr)   /* 数据元说明 / 报表名称值 */
  minmax(96px, 0.95fr)    /* 表 / 数据范围标签 */
  minmax(72px, 0.75fr)    /* 字段 */
  minmax(140px, 1.35fr)   /* 逻辑 / 数据范围内容 */
```

## 6. 验收标准（AC）

### P0 — MVP 必须满足

| 编号 | 验收标准 |
|------|----------|
| AC-01 | 输入关键词后 300ms 内展示联想列表 |
| AC-02 | 联想含数据项名称、表名、报送项目，最多 10 条，包含匹配 + 相关度排序 |
| AC-03 | 搜索响应 < 2 秒（万级数据） |
| AC-04 | 仅显示有命中结果的报送标签页 |
| AC-05 | 标签页内按报表名称分块展示 |
| AC-05a | 组头左侧：报表名称 + 采集范围（表级） |
| AC-05b | 组头右侧：数据范围 = 取数表 + 关联条件（表级） |
| AC-05c | 「数据范围」第 3 列与「表」第 3 列左缘对齐 |
| AC-06 | 数据表格五列：数据项名称、数据元说明、表、字段、逻辑 |
| AC-07 | 左右区域颜色一致，无分区着色、无关键词颜色高亮 |
| AC-08 | 单元格内边距、行高、边框符合 §4.2 规范 |
| AC-09 | 支持 Excel 导入一表通数据 |
| AC-10 | 断网状态下所有功能可用 |
| AC-11 | BA 可在 30 分钟内完成本地环境搭建 |

### P1 — 第二版

| 编号 | 验收标准 |
|------|----------|
| AC-20 | 支持 EAST、1104、金数等全部报送类型 |
| AC-21 | 报送配置化，新增报送仅加表 + 配置 |
| AC-22 | 搜索历史记录 |
| AC-23 | 导出检索结果为 Word/PDF |

### P2 — 桌面端

| 编号 | 验收标准 |
|------|----------|
| AC-30 | 打包为 Windows exe，双击启动 |
| AC-31 | 系统托盘 + 快捷键唤起搜索 |

## 9. 与现有代码的差异（待迁移）

当前仓库使用 `meta_tables` / `meta_fields`（Excel 总表结构），与 Spec 中的多报送分表模型（`YBT_ZL` 等）不一致。开发时需：

1. 新增 `report_config` 与 `YBT_ZL` 表结构
2. 重写 `search` 服务：按报送聚合、按表名分组
3. 重写 `SearchView.vue`：标签页 + 五列网格结果块
4. 新增联想 API 与前端下拉组件
5. 更新 `import-excel.js` 列映射
